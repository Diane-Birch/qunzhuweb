参考网页https://www.yytthm.cn/ ，搭建一个哈尼梯田红米文化展示类网站，核心要求为前端所有动态内容（含轮播图、各板块标题 / 文字 / 配图、产品信息等）均不硬编码，需配套管理员后台管理系统，支持后台登录后自主编辑、发布、修改前端所有展示内容。项目完成后需生成标准化 Readme.md 文件（适配 GitHub 仓库发布），并保证代码可打包部署。具体开发要求如下：
1. 技术栈指定
（1）前端：基于 Vue3 + Element Plus 组件库 开发，保证代码规范、易维护；页面需做全端自适应适配（兼容 PC 端主流分辨率、平板 / 手机移动端），Element Plus 组件样式需贴合 “原生态农耕文化” 调性做轻量化定制（避免纯默认样式）；（2）后端：基于 FastAPI 框架 开发，严格遵循 RESTful API 接口设计规范，实现前后端分离；数据存储使用 MySQL 数据库，支持数据的增删改查、分页、排序；（3）核心交互：前后端通过 JSON 格式交互，接口需包含身份验证（如 Token 登录）、参数校验、异常处理，保证数据安全和请求稳定性；
2. 整体风格与前端要求
（1）风格：贴合原网站的原生态、农耕文化调性，视觉上保留原站的板块布局逻辑（如首页核心卖点区、产品介绍区、企业动态区、产业振兴区等核心板块），无需完全复刻，风格统一即可；（2）动态内容：所有可展示内容（轮播图的图片 / 文案、各板块的标题 / 正文 / 配图、产品参数 / 介绍、新闻动态等）均为后台可配置，前端仅通过 FastAPI 提供的 RESTful 接口渲染数据，无任何固定写死的展示内容；
3. 后台管理系统核心功能
（1）基础功能：管理员账号密码登录（基于 Token 验证）、权限唯一（仅管理员可操作），后台界面基于 Element Plus 组件搭建，保持和前端风格统一；（2）内容管理：支持轮播图的上传 / 删除 / 排序 / 文案编辑，各板块标题 / 文字 / 配图的新增 / 修改 / 保存，产品信息（如红米参数、品种）的编辑 / 更新，企业动态新闻的发布 / 编辑 / 删除；（3）操作体验：后台界面简洁易懂，操作流程轻量化（编辑后一键保存生效，前端实时同步），表单、弹窗、上传等交互组件复用 Element Plus 原生能力并优化体验；
4. 交付要求
（1）代码规范：所有代码附带清晰的注释（关键函数 / 接口 / 配置需详细说明），前后端代码按模块拆分（前端分 views/components/api，后端分 router/schema/crud/models），目录结构符合 GitHub 开源项目规范；（2）打包部署：提供前端打包（npm run build）、后端打包（依赖冻结）的脚本 / 说明，保证代码可直接打包后上传至 GitHub 仓库；（3）Readme.md 生成（核心）：
必含模块：项目介绍、技术栈清单、环境要求（前端 Node.js 版本 / 后端 Python 版本 / MySQL 版本）、本地部署步骤（前端安装依赖 / 后端配置环境变量 / 数据库迁移 / 启动服务）、目录结构说明、核心接口文档链接、常见问题排查；
格式要求：使用 Markdown 规范排版（含标题层级、代码块、列表、链接），语言简洁易懂，适配 GitHub 网页端展示效果；
